# How To Use Research Before A Pick

This is the practical workflow before tracking or betting anything.

## Step 1: Check the market

Use app tabs:

- Soccer
- MLB
- Value
- Algorithm
- Bankroll Watch

Look for:

- best available line,
- DraftKings line if using the free $200 workflow,
- model probability,
- break-even probability,
- edge,
- fair line,
- stale-line warnings.

## Step 2: Check sport-specific context

For Soccer:

- injury report,
- suspensions,
- starting XI,
- rotation/rest,
- travel,
- venue/altitude/weather,
- draw vs to-advance market difference.

For MLB:

- starting pitcher,
- confirmed lineup,
- bullpen fatigue,
- weather/wind/roof,
- park factor,
- previous-game result,
- after-loss prior,
- Statcast rolling form.

## Step 3: Check Intel

Open [[Injury and Lineup Intel]] in the app.

Add a note for every meaningful item.

Use:

- `High` for star/player/pitcher availability that can move the line.
- `Medium` for lineup or weather context.
- `Low` for mild context.

Status:

- `Watch`: not confirmed yet.
- `Confirmed`: active adjustment.
- `Cleared`: no longer applies.

## Step 4: Re-check Algorithm

After adding Intel:

- go back to Algorithm,
- check if model probability changed,
- check if source note shows an Intel adjustment,
- confirm edge still exists.

## Step 5: Compare with Bankroll Watch

Bankroll Watch is a watchlist.

It is not an order.

Only consider a pick if:

- it clears the probability floor,
- price is still current,
- Intel is checked,
- DraftKings line is available if that is the intended book,
- stake label is not `Watch only`.

## Step 6: Track it

If it survives:

- add/track in Model Lab,
- record price,
- record probability,
- record stake,
- later grade win/loss/draw,
- compare closing line value.

Use:

The model should get stricter where it has historically failed.
